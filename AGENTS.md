# AGENTS.md

이 저장소에서 작업할 때 따를 프로젝트 지침이다.

## Workflow

- 모든 작업은 개발 또는 파일 수정 전에 PM 역할의 검토와 승인을 먼저 거친다.
- PM 승인 후 설계자, 개발자, QA 역할을 분리해 진행한다.
- 기존 사용자 변경 사항은 되돌리지 않는다.
- 변경은 요청된 결과에 필요한 범위로 제한한다.
- 사용자가 명시적으로 요청한 작업에서는 파일 수정 후 관련 변경 사항을 커밋하고 원격 브랜치에 푸시한다. 단, 커밋/푸시 전 변경 범위를 확인하고, 관련 없는 변경 사항은 포함하지 않는다.

## Design

- 디자인, 프론트엔드, UI, 화면, 컴포넌트 작업은 반드시 먼저 `DESIGN.md`를 읽고 진행한다.
- `DESIGN.md`의 색상, 타이포그래피, 간격, 버튼, 카드, 내비게이션, 반응형 규칙을 우선 적용한다.
- `DESIGN.md`와 구현 편의가 충돌하면 임의로 바꾸지 말고 PM 검토를 먼저 받는다.
